核心功能定义
开发3D 网络关联图谱，对标 Obsidian 笔记思维导图 / 图谱模式，以三维可视化形式展示情侣所有日常记录，实现数据联动、直观浏览、情感关联。
核心设计理念
节点 = 单条日记记录（包含文字、图片、创建时间）
连线 = 共同标签（相同标签的日记节点自动生成连接）
三维空间 = 所有日记的全局关联网络
风格 = 温馨、浪漫、简约、现代，贴合情侣日记场景
详细功能开发需求
1. 3D 节点设计
每一条日记 = 一个球形 / 立方体 3D 节点
节点视觉区分：
大小：根据日记内容长度 / 图片数量动态调整
颜色：使用情侣主题色（粉色、蓝色、紫色渐变）
悬停效果：放大、发光、显示日记预览
节点固定展示信息：创建时间、核心标签缩略
2. 3D 连线设计
标签驱动连线：两个节点拥有任意相同标签，自动生成柔性连线
连线样式：半透明、发光、柔和渐变，不遮挡节点
连线逻辑：相同标签越多，连线越粗 / 越亮
3. 核心交互功能（必须实现）
相机控制
鼠标拖拽：旋转 3D 空间视角
滚轮：缩放视图
双击节点：快速聚焦 / 定位到该节点
节点交互
单击节点：查看完整日记内容（弹窗 / 侧边栏）
悬停节点：高亮该节点 + 所有关联连线 + 关联节点
筛选功能
按标签筛选：点击标签 → 只显示对应节点与连线
按时间筛选：近 7 天 / 近 30 天 / 全部记录
全局控制
自动旋转：图谱缓慢自转（默认开启，可关闭）
重置视图：一键回到初始视角
全屏模式
4. 与 AI 标签提取功能的联动规则
新增日记 → AI 生成标签 → 自动同步到 3D 图谱
新增标签 → 自动匹配所有节点，实时生成新连线
编辑标签 → 图谱自动更新连线关系
标签为图谱唯一连接依据，无标签节点独立存在
四、UI/UX 视觉设计要求（情侣专属风格）
背景：深色半透明渐变 + 星空 / 粒子柔光效果（不刺眼）
节点：磨砂发光质感，柔和不尖锐
连线：荧光渐变，动态流光效果
整体氛围：温馨、治愈、科技感适中，拒绝冰冷硬核风格
响应式：适配电脑端，保证流畅运行
五、数据结构与数据库联动要求
读取现有数据库表：
diaries 表：id、内容、图片、创建时间
tags 表：标签名称、关联日记 id
前端通过 API 获取全量日记 + 标签数据，实时构建 3D 图谱
数据更新后，图谱无刷新自动重新渲染
六、Coding Agents 开发交付标准
代码可直接集成到现有 Next.js 项目，无冲突、无冗余
封装为独立组件：<Diary3DGraph />，可直接路由跳转使用
注释清晰，结构模块化，便于后续迭代
性能优化：节点数量过多时自动简化渲染，保证流畅
自带基础控制面板（视角、筛选、自动旋转开关）
